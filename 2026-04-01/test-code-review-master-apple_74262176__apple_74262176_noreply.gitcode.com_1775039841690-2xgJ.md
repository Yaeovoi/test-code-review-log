*   **缺乏完整性校验**：直接通过 `wget` 下载二进制文件存在供应链安全风险。如果发布源被篡改或网络传输出现问题，可能导致 CI 流程执行恶意代码或运行损坏的文件。
    *   **建议**：在下载后增加校验步骤（如 SHA256 校验），或者使用 GitHub 的官方 Action（如 `actions/download-artifact` 或专门的 Release 下载 Action），以确保依赖包的完整性和来源可信。
